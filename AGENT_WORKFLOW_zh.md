# 🤖 个人文献处理助手指南 (Agent Workflow)

这是一套轻量级的个人文献阅读与总结工作流，旨在指导 AI 编程助手（如 Codex、Cursor、Copilot）处理由 MinerU 提取的学术论文。

## 📌 目标说明
作为 AI 助手，你的任务是读取 MinerU 提取的长篇 Markdown 论文，将其转换为一份结构清晰、基于第三人称视角的学术阅读笔记，并确保文中引用的图表链接绝不造假。

## 📁 目录约定
- **输入目录**: `data/input/{paper_folder}/` (内含论文原始的 `full.md` 及提取的 `images/` 图像文件夹)
- **Markdown 输出**: `data/output/{paper_id}_summary.md`
- **Word 输出**: `data/output/{paper_id}_summary.docx`

## 🛠️ 执行步骤 (Agent Step-by-Step)
当接收到用户的文献处理指令时，请按顺序执行：

1. **阅读输入**: 
   - 提取并阅读目标论文夹下的 `full.md` 文件。
   - 通过 `ls` 或读取目录结构，了解 `images/` 下真实存在的图片文件名，以建立可靠的图像映射概念。
2. **生成中文结构化总结**:
   - **标题**: 第一行必须保留英文原题，不得改写或压缩；第二行提供忠实、专业的中文译名。凝练要求只适用于正文小节标题，不适用于论文原题。
   - **摘要**: 对原文的 Abstract 摘要部分进行直接、忠实的中文学术翻译。保证原汁原味，不能随意删减骨架。唯一且最重要的一定是：在翻译时，严格将所有的第一人称表述（如我们、本文、本研究）替换为第三人称陈述（如该研究、文章、科研作者等）。
   - **研究方法**: 详实而专业地描述核心研究框架、模型架构或实验设计。如果在原文档中找到了实验装置、流程图或公式图的对应图片，必须将其链接到这一段落辅助说明并辅以图注。
   - **主要内容/核心发现**: 压缩为不超过 4 个逻辑小节，小节标题应凝练，原则上不超过 18 个汉字。不要机械执行“一节一图”，每个小节可以包含多张逻辑相关的图表，只要它们共同支撑同一论点。**绝对禁止使用口语化或夸张的修辞手法**（如“另辟蹊径”、“别出心裁”、“砍掉一半”等），必须全程使用极其严谨、客观、科学的学术界标准书面用语。**每一节的字数必须极尽详实，要求每节至少扩展至 500 个汉字左右**，深入拆解模型内部机制、定量证据与对比实验分析。高度依赖并引用相关结果图表。
   - **结论**: 末尾只保留一个 `结论` 小节。不要单独写 `环境意义`，也不要使用 `精读结论` 或 `精度结论` 作为标题。结论部分重在概括概念贡献与机制认识，不重复罗列预测精度指标。
   - **排版风格**: 内容必须充实丰富、段落完整饱满，完全对标高质量顶刊文献的精读长文的标准。拒绝简单的要点罗列。
3. **Markdown 输出落盘**: 将最终整合完成的 Markdown 以 `{paper_id}_summary.md` 的命名保存到 `data/output/`。
4. **转换为 Word**: Markdown 校验通过后，运行 `python src/md_to_docx.py {paper_id}`，基于 `data/example format/` 下的参考模板生成 `{paper_id}_summary.docx`。
5. **校验 Word**: 运行 `python src/check_docx.py {paper_id}`，确保标题、DOI 与图片插入均已替换为当前论文内容。

## ⚠️ 绝对避免的错误 (Mistakes to Avoid)

这关乎总结的可用性，请严格遵守！
1. **禁止捏造图表路径 (No Hallucinated Images)**: 
   - 总结中引用的任何图片格式 `![图名](images/xxx.jpg)`，其对应路径必须**百分之百在原始的 `images/` 目录中真实存在**。
   - 如果找不到符合的图片，请直接纯文本描述，**绝不能**生造诸如 `images/figure1.jpg` 这样虚假的路径名。
2. **执行严格的第三人称学术陈述 (Enforce 3rd-Person Narration)**: 
   - 原文常会有 "we found", "our study", "we propose"。
   - 翻译和改写时，**严禁使用**："我们发现"、"我们的方法"、"本研究"、"本文"。
   - **必须替换为**："该研究发现"、"所提方法"、"科研团队指出"、"该文章"。
3. **禁止 Word 模板残留**:
   - 生成的 Word 文档不得保留参考模板原论文的题目、DOI、正文内容、图像、封面图或隐藏媒体。
   - 不自动生成封面；如果 MinerU 未提供可用封面，则封面位置留空，供后续人工添加。
   - Markdown 中的图片应按原位置插入 Word，并使用 Markdown 图注作为居中的 Word 图注。
   - 如果 MinerU 输入中存在可靠的图文摘要图片，应将其插入 Word 的中文摘要之后。
